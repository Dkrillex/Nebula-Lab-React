# Assets 资源文件文档

本目录包含项目中使用的静态资源文件，包括图片、视频、音频等演示文件。

## 目录结构

```
assets/
└── demo/
    ├── 1111.png              # 演示图片
    ├── ec6-4dbbffde26e2.mp4  # 演示视频
    ├── file_example_MP3_700KB.mp3  # 演示音频文件
    ├── productImage.png       # 产品图片
    └── userFaceImage.png      # 用户头像图片
```

## 文件说明

### demo/ 目录
包含项目中使用的演示和示例文件：

- **1111.png**: 通用演示图片
- **ec6-4dbbffde26e2.mp4**: 演示视频文件
- **file_example_MP3_700KB.mp3**: 音频文件示例（700KB）
- **productImage.png**: 产品相关图片，用于产品展示、数字人视频等功能
- **userFaceImage.png**: 用户头像图片，用于用户界面、数字人功能等

## 使用方式

### 在组件中引用资源
```typescript
// 方式1: 直接导入
import productImage from '@/assets/demo/productImage.png';
import userFaceImage from '@/assets/demo/userFaceImage.png';

function MyComponent() {
  return (
    <div>
      <img src={productImage} alt="Product" />
      <img src={userFaceImage} alt="User" />
    </div>
  );
}

// 方式2: 使用 public 目录（如果文件在 public 目录下）
// <img src="/demo/productImage.png" alt="Product" />
```

### 在 CSS 中引用
```css
.background {
  background-image: url('@/assets/demo/productImage.png');
}
```

## 注意事项

1. **文件大小**: 注意资源文件的大小，过大的文件会影响页面加载速度
2. **文件格式**: 
   - 图片：推荐使用 PNG（透明背景）或 WebP（更小体积）
   - 视频：推荐使用 MP4 格式
   - 音频：推荐使用 MP3 格式
3. **路径引用**: 使用 `@/assets/` 别名引用资源，确保构建工具正确处理
4. **版本控制**: 资源文件应纳入版本控制，但注意不要提交过大的文件
5. **CDN 部署**: 生产环境建议将静态资源部署到 CDN，提高加载速度

## 与 public/ 目录的区别

- **assets/**: 通过构建工具（Vite）处理，会被打包和优化，支持导入语法
- **public/**: 直接复制到输出目录，不会被构建工具处理，使用绝对路径引用

## 添加新资源

1. 将资源文件放入 `assets/` 或 `assets/demo/` 目录
2. 在代码中使用 `import` 导入资源
3. 确保文件大小合理，避免影响性能
4. 更新本文档，记录新增的资源文件
